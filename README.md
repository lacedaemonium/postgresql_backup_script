# PostgreSQL Backup Script (BASH)

Скрипт для автоматического резервного копирования баз данных PostgreSQL.

## Основные возможности
- Автоматическое определение списка всех баз данных на сервере.
- Создание дампов (`pg_dump`), сжатие в `gzip` и проверка целостности архивов.
- Логирование каждого этапа работы (успех/ошибка) в лаконичном формате.
- Безопасность: автоматическая очистка временных файлов и дампов при любых сбоях или завершении работы.
- Перенос готовых бэкапов в выделенный каталог `/backups`.

## Требования
- Установленные утилиты: `psql`, `pg_dump`, `gzip`.
- Права на запись в каталог `/backups` и в папку со скриптом (для лога).

## Настройка
Перед запуском отредактируйте блок `НАСТРОЙКИ` в теле скрипта:
- `DATABASE_HOST`: IP или имя хоста сервера БД.
- `DATABASE_BACKUP_USER`: пользователь с правами на чтение баз.
- `DATABASE_BACKUP_USER_PASSWORD`: пароль пользователя.
- `BACKUP_DIR`: путь к директории для хранения бэкапов (опционально).

## Запуск
1. Сделайте скрипт исполняемым:
   ```bash
   chmod +x postgres_backup.sh
   ```

2. Запуск:
   ```bash
   ./postgres_backup.sh
   ```

## Cледует учитывать
что пользователь, который указан в DATABASE_BACKUP_USER, должен иметь права на чтение/выгрузку таблиц из PostgreSQL.
Соответствующие права можно выдать с помощью команд:
```
GRANT USAGE ON SCHEMA public TO DATABASE_BACKUP_USER;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO DATABASE_BACKUP_USER;
GRANT SELECT ON ALL SEQUENCES IN SCHEMA public TO DATABASE_BACKUP_USER;
```
заменив DATABASE_BACKUP_USER на имя пользователя и выполнив команды поочередно, подключившись в каждую из баз данных, которую планируется бэкапить.
